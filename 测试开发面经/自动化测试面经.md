### 2、Playwright遇到弹窗怎么定位？
Playwright 可以使用`page.on("dialog", handler)`来监听弹窗的`dialog`事件，从而实现对弹窗的处理和定位。`dialog`对象具有一些属性和方法，可用于获取弹窗相关信息或执行操作，例如`dialog.accept()`用于点击弹窗上的确定按钮，`dialog.dismiss()`用于点击取消按钮，`dialog.message`用于获取对话框中显示的消息，`dialog.type`用于返回对话框的类型（如`alert`、`beforeunload`、`confirm`或`prompt`）。
### 3、playwright遇到内嵌framwork怎么定位？
在 Playwright 中处理内嵌框架（iframe）时，可以使用 `page.frame_locator()`方法创建一个框架定位器（frame locator）来捕获 iframe 中检索和定位元素
locator = page.frame_locator("my-frame").get_by_text("submit") locator.click()

### 4、怎么集成到CI上的
使用 Docker 将 Playwright 集成到 GitHub Actions 上，可以按照以下步骤进行操作：
1. 在 GitHub 仓库的根目录下，创建一个名为`.github/workflows`的文件夹（如果该文件夹不存在）。
2. 在`.github/workflows`文件夹中，创建一个 YAML 文件，例如`playwright.yml`，用于定义 GitHub Actions 的工作流程。
3. 以下是一个简单的`playwright.yml`示例，展示了如何使用 Docker 中的 Playwright 镜像来运行测试：
```ymal
name: Playwright Tests

on:
  push:
    branches: (main, master)
  pull_request:
    branches: (main, master)

jobs:
  playwright:
    name: 'Playwright tests '
    runs-on: ubuntu-latest
    container:
      image: mcr.microsoft.com/playwright:v1.44.1-jammy 
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: 18
      - name: install dependencies
        run: npm ci
      - name: run your tests
        run: npx playwright test
      - name: publish report
        if: ${{!cancelled()}}
        uses: actions/upload-artifact@v4
        with:
          name: playwright-report
          path: playwright-report/
          retention-days: 30
```

### 5、参与了自动化测试架构的哪方面

1. 测试用例设计与规划
    - 确定要覆盖的功能、场景（10个）和边界条件，设计有效的测试用例结构和流程。
2. 选择合适的定位策略
    - 如前面所述，根据页面元素的特点和测试需求，选择最优的元素定位方法（用户可见元素），确保测试的准确性和稳定性。
3. 页面交互模拟（PO模式）
    - 编写代码来模拟用户在页面上的各种操作，如点击、输入、下拉选择等。
4. 断言与验证
    - 定义断言来验证页面的状态、元素的属性、文本内容等是否符合预期结果。
5. 数据驱动测试
    - 构建数据集合，实现测试用例可以根据不同的数据输入进行执行，提高测试的覆盖度。
6. 错误处理与重试机制（try-except)
    - 处理在测试过程中可能出现的错误，设置合理的重试策略，以应对暂时的网络问题或不稳定的页面响应。
7. 测试环境搭建与配置
    - 配置测试运行所需的环境，包括浏览器类型、版本，以及可能的代理设置等。
8. 测试报告与日志
    - 设计有效的测试报告格式，记录测试过程中的关键信息和结果，便于问题追踪和分析。同时，完善日志记录，方便调试和问题排查。
9. 框架集成与扩展
    - 将 Playwright 与现有的测试框架（如 Jest、Mocha 等）进行集成，或者根据项目需求扩展 Playwright 的功能。
10. 性能优化
    - 对测试脚本进行性能优化，减少测试执行时间，提高测试效率。

### 12、异步和同步的区别，怎么实现

● 同步和异步的区别
同步：多个任务需要按照一定的顺序执行，一个任务未收到相应就会阻塞进程
异步：多个任务可以相互独立的完成，不会阻塞主线程，可以提高响应速度
● 怎么实现
使用async定义异步函数，await紧跟promise表达式

### 18、如何设置工作线程
playwright默认并行运行测试，它使用工作线程运行测试，可用的CPU核心数决定了工作线程数，playwright使用一半的可用CPU核心
```
module.exports=defineConfig({ testDir:"./tests", workers:"5%", projects:[ { name:"chromium", use:{...devices["Desktop Chrome"]}, }, ], }); 
```

测试将在拥有的5%的核心数上运行，比如拥有4个CPU核心，那么设置在0.2个CPU核心上运行，一个CPU支持一个一个线程的前提下，所以将只有一个工作线程

### 19、playwright怎么实现图片验证码
`Playwright` 本身并不直接提供处理图片验证码的内置功能。但是，您可以结合一些图像处理库（如 `Pillow` ）和验证码识别服务（如云打码等）来实现处理图片验证码。
以下是一个大致的步骤：
1. 使用 `Playwright` 抓取包含图片验证码的页面。
2. 获取图片验证码的 `src` 属性值或直接获取图片元素的截图。
3. 将获取到的图片数据传递给图像处理库进行处理，例如裁剪、灰度化、二值化等操作，以提高识别准确率。
4. 使用验证码识别服务或自己训练的模型来识别处理后的图片。
5. 根据识别结果，在页面中填写验证码并提交表单。
## python-appium app UI自动化
### 1、APP端自动化测试中，遇到动态ID怎么解决
1. **根据其它属性定位**：检查元素是否具有其他固定且唯一的属性，例如 name、class、text 等，并使用这些属性进行定位。
```python
1.driver.find_elements(By.id(" android:id/content ")).click();
2. driver.find_elements(By.name("5")).click();
3.driver.find_elementss(By.className("android.widget.Button"));
4.driver.find_elements(By.xpath("//android.widget.TextView[contains(@resource-id,'com.android.mms:id/action_compose_new')]")).click();
5.driver.find_element(by=By.ACCESSIBILITY_ID, value="equals")
6.driver.find_element(by=AppiumBy.ANDROID_UIAUTOMATOR, value='new UiSelector().text(\"9\")’).send_keys("D8HAIYZ1")
```
3. **根据相对关系定位**：通过父子、兄弟等相邻节点的关系来定位元素。
```python
# 父元素定位
driver.find_element_by_android_uiautomator('resourceId("com.baidu.yuedu:id/rl_tabs")')
# 先找到父亲，再找儿子
driver.find_element_by_android_uiautomator('resourceId("com.baidu.yuedu:id/rl_tabs").childSelector(text("小说"))')

#兄弟关系fromParent
# **根据子节点定位父节点**：使用`fromParent`方法，有时候父元素不好定位，但是跟他相邻的兄弟元素很好定位，这时候就可以通过兄弟元素，找到同一父级元素下的子元素。
driver.find_element_by_android_uiautomator('resourceId("com...").fromParent(text("小说"))')

# 通过弟弟节点定位哥哥节点
driver.find_element_by_xpath("//div[@id='二哥节点']/preceding-sibling::div[1]")
# 通过哥哥节点定位弟弟节点
driver.find_element_by_xpath("//div[@id='二哥节点']/following-sibling::div[1]")
```

### 1、用Jmeter做了什么？
用jmeter做接口测试、并发测试，如：调用接收短信并生成线索的接口，模拟同一时间接受到大量短信（短信内容相同、短信内容不同、多个用户接收到同一条短信）

**了解线程组**
在线程组中，可以设置并发用户数、循环次数等参数。
1、设置线程数（并发用户数）：在线程组中，设置"线程数"为需要的并发用户数。
2、设置循环次数：在线程组中，设置"循环次数"为需要执行的循环次数，如果不设置循环次数，则默认为1。

### 2、Jmeter和postman做接口测试的区别？

1. 性能测试能力：
    - Jmeter 主要用于性能测试，可以模拟高并发、大流量的场景，评估接口在不同负载下的性能表现，如响应时间、吞吐量等。
    - Postman 更侧重于接口的功能测试，虽然也能进行简单的并发测试，但性能测试方面的功能相对较弱。
2. 测试场景设置：
    - Jmeter 可以方便地设置复杂的测试场景，如多个线程组、循环、调度器等，以模拟真实的用户行为和负载模式。
    - Postman 更适合简单的单个接口测试和少量接口的组合测试。
3. 数据驱动：
    - Jmeter 支持从外部数据源（如 CSV 文件、数据库等）读取数据进行参数化，方便进行大量不同数据的测试。
    - Postman 也支持参数化，但在处理大规模数据驱动测试时可能不如 Jmeter 灵活。
4. 结果分析：
    - Jmeter 提供了丰富的结果分析图表和报告，如聚合报告、图形结果等，有助于全面分析性能数据。
    - Postman 的结果分析相对简单，主要侧重于接口的响应状态和数据展示。
5. 团队协作：
    - Postman 有较好的团队协作和版本控制功能，方便团队成员之间共享和管理接口测试用例。
    - Jmeter 在团队协作方面相对较弱，需要借助其他工具来实现版本控制和团队共享。
6. 脚本编写：
    - Jmeter 使用基于 Java 的脚本语言，对于不熟悉 Java 的用户可能有一定的学习成本。
    - Postman 则使用相对简单直观的界面操作和 JavaScript 脚本（在 Postman 脚本中），上手相对容易。
如果需要对一个大型系统的接口进行全面的性能评估和压力测试，Jmeter 可能是更好的选择。而如果您主要是进行接口的功能验证和初步的调试，Postman 会更加便捷高效。
